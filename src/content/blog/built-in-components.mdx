## Step Indicator

- **Location:**
  - `src/components/post/Steps/Steps.astro`
  - `src/components/post/Steps/Step.astro`
- **Import path:**
  - `@shComponents/Steps/Steps.astro`
  - `@shComponents/Steps/Step.astro`
- **Props:**
  | Component | Prop name | Type | Description |
  | --- | --- | --- | --- |
  | `<Steps />` | `start` | number | Sets the starting step number of the step indicator. Default is 1. |
  | `<Steps />` | `class` | string | Additional CSS class for customizing step indicator styles. |
  | `<Step />` |`title` | string | The step title displayed in the step indicator. |

You can use the `Steps` and `Step` components to create a step indicator, which is useful for showing workflows or guides.

```astro
<Steps start={3}>
  <Step title="Step One">This is the description for step one.</Step>
  <Step title="Step Two">This is the description for step two.</Step>
  <Step title="Step Three">
    This is the description for step three. You can add more content here to explain this step in detail.
    <ol>
      <li>This is a list item. Yes, you can use lists.</li>
      <li>But you need to add HTML tags manually.</li>
    </ol>
  </Step>
</Steps>
```

This renders a step indicator with three steps, each with a title and description. You can add more steps as needed or customize each step's content.

## File Tree

- **Location:**
  - `src/components/post/Files/Files.astro`
  - `src/components/post/Files/Folder.astro`
  - `src/components/post/Files/File.astro`
- **Import path:**
  - `@shComponents/Files/Files.astro`
  - `@shComponents/Files/Folder.astro`
  - `@shComponents/Files/File.astro`
- **Props:**
  | Component | Prop name | Type | Description |
  | --- | --- | --- | --- |
  | `<Files />` | `path` | string | Sets the root path of the file tree (optional). |
  | `<Files />` | `class` | string | Additional CSS class for customizing file tree styles. |
  | `<Folder />` | `name` | string | Folder name. |
  | `<Folder />` | `defaultOpen` | boolean | Sets whether the folder is expanded by default (optional, default is `False`). |
  | `<Folder />` | `highlighted` | boolean | Sets whether the folder is highlighted (optional, default is `False`). |
  | `<File />` | `name` | string | File name. |
  | `<File />` | `icon` | FileIconType | Sets the file icon type (default is `normal`; see options below). |
  | `<File />` | `highlighted` | boolean | Sets whether the file is highlighted (optional, default is `False`). Note that highlighting a file does not automatically highlight its parent folder. To highlight a folder, set the `highlighted` prop on `<Folder />`. |

```astro
<Files path="/sh-blog-next/">
  <Folder name="app" defaultOpen>
    <File name="layout.tsx" icon="code" />
    <File name="page.tsx" />
    <File name="global.css" highlighted />
  </Folder>
  <Folder name="components" highlighted>
    <File name="button.tsx" icon="code" />
    <File name="tabs.tsx" />
    <File name="dialog.tsx" highlighted />
    <Folder name="core">
      <File name="Header.astro" />
      <File name="Footer.astro" />
    </Folder>
  </Folder>
  <File name="package.json" />
</Files>
```

## GitHub Card

- **Location:** `src/components/GHCard/GitHubCard.astro`
- **Import path:** `@shComponents/GHCard/GitHubCard.astro`
- **Props:**
  | Prop name | Type | Description |
  | --- | --- | --- |
  | `repo` | string | Specifies which GitHub repository to display, in the format `username/repository` or `https://github.com/username/repository`. |
  | `class` | string | Additional CSS class for customizing GitHub card styles. |

:::danger[Warning]
To use the GitHub Card component, you must set a valid [GitHub Personal Access Token](https://github.com/settings/personal-access-tokens) in the `GITHUB_TOKEN` environment variable.
:::

## Admonition Blocks

> Admonition blocks are already integrated into the Markdown parser, so you can create them directly using Markdown syntax.

```markdown
:::note[Note]
Highlights information that users should take into account, even when skimming.
:::

:::tip[Tip]
Optional information to help a user be more successful.
:::

:::important[Important]
Important information for a user to know.
:::

:::danger[Danger]
Serious and/or dangerous situations.
:::

:::caution[Caution]
Negative potential consequences of an action.
:::
```

:::

:::tip[Tip]

:::

:::important[Important]

:::

:::danger[Danger]
